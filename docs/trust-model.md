# Trust Model

Runcap must not become another confident AI system that invents progress.

## Truth Labels

Every important value needs a source:

- `observed`: measured from local execution, git diff, exit code, terminal output, files.
- `provider_usage`: token usage returned by the upstream model provider.
- `mock_provider_usage`: generated by mock gateway for demos only.
- `calculated_from_static_price_table`: cost estimated from known token usage and a prototype price table.
- `manual_calibration`: user-visible subscription percentage before/after a mission.
- `budget_guard`: call blocked by a local budget rule.
- `unknown`: the system cannot honestly know.

## What the Product Can Know

For CLI-wrapped coding missions it can observe:

- command;
- working directory;
- terminal output;
- exit code;
- git diff before/after;
- changed files;
- parsed error patterns;
- verification command result.

## What the Product Cannot Know Yet

It cannot reliably know:

- exact Claude Max / ChatGPT subscription burn unless the user calibrates visible percentages;
- quality of every output without domain-specific proof;
- work performed by agents launched outside the wrapper;
- exact pricing for every model without a verified live price catalog.

## Design Rule

If a number or recommendation is not backed by evidence, the product must say so.

The product should prefer:

```text
Unknown, enable calibration.
```

over:

```text
Estimated with fake precision.
```

## Rescue Rule

A rescue recommendation is valid only if it includes:

1. observed failure signal;
2. source file, command, or config clue when available;
3. confidence level;
4. smallest next action;
5. a prompt that narrows the next agent run.
